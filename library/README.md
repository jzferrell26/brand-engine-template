# Library

Documentation for this client's brand engine instance: requirements (PRDs/IRDs), narrative knowledge, QA reports, and notes. Structure follows the `library-guardian` convention shared across the Guild.

- `requirements/` -- forward PRDs (backlog/in-work/completed) and reports.
- `knowledge/` -- narrative architecture and standards docs (private + public).
- `qa/` -- QA reports (authored only by `quality-guardian`).
- `notes/` -- human-only scratch space; no agent writes here.
- `issues/` -- ingested issues (backlog/in-work/completed).
